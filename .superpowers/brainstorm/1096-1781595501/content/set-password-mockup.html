Behaviour notes
  • Rules checklist updates live as the user types — no submit needed to see feedback
  • Eye icon toggles password visibility on each field independently
  • "Set password" button is disabled until: length valid + 3 of 4 rules met + both fields match
  • On success: calls supabase.auth.updateUser({password}) + sets users.password_set = true → redirects to /timesheet/daily
  • Bilingual (he/en) — same RTL/LTR logic as existing login page
Password complexity rule
Must satisfy at least 3 of these 4:
  1. ≥1 uppercase letter [A-Z]
  2. ≥1 lowercase letter [a-z]
  3. ≥1 number [0-9]
  4. ≥1 special character [!@#$%…]
Length: 8–16 characters
Guard: unauthenticated access
Middleware allows authenticated users through to /login/set-password (they need a live session to call updateUser). Unauthenticated visitors are redirected to /login.
